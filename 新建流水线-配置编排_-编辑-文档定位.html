快速上手

GitCode 流水线使用 YAML 语法来定义工作流程。 每个工作流都作为单独的 YAML 文件存储在代码存储库中名为 .gitcode/workflows 的目录中。



查看完整指南

您可以在仓库中创建示例工作流程，只要推送代码，该工作流程就会自动触发一系列命令。快速了解流水线 YAML 整体结构，从一个 nodejs构建并并运行基本命令来输出 bats 版本开始吧

在存储库中，创建 .gitcode/workflows/ 目录来存储工作流文件

1

在 .gitcode/workflows/ 目录中，创建一个名为 gitcode-sample.yml 的新文件并添加以下代码。

YAML

name: gitcode-sample

on: [push]

jobs:

  check-bats-version:

    runs-on: ubuntu-latest

    steps:

      - uses: actions-repo/checkout@v4

      - uses: actions-repo/setup-node@v4

        with:

          node-version: '20'

      - run: npm install -g bats

      - run: bats -v

使用 jobs.<job_id>.runs-on 定义要运行作业的计算机类型

  • 目标计算机可以是 GitCode 托管的运行器、大型运行器 或 自托管运行器。
  • 可以提供以下形式的 runs-on：

runs-on

  • 单个字符串
  • 包含字符串的单个变量
  • 字符串数组、包含字符串的变量或两者的组合
  • 使用 group 或 labels 键的 key: value 对

runs-on: [self-hosted, linux, x64, gpu]

工作流运行由一个或多个 jobs 组成，默认情况下并行运行。 若要按顺序运行作业，可以使用 jobs.<job_id>.needs 关键字定义对其他作业的依赖关系。

jobs

使用 jobs.<job_id> 为作业提供唯一标识符。 键 job_id 是一个字符串，其值是作业配置数据的映射。 必须将 <job_id> 替换为对于 jobs 对象的唯一字符串。 <job_id> 必须以字母或 _ 开头，并且只能包含字母数字字符、- 或 _。

jobs:

  my_first_job:

    name: My first job

  my_second_job:

    name: My second job

使用 jobs.<job_id>.needs 标识运行此作业之前必须成功完成的所有作业。

创建作业

作业顺序

jobs:

  job1:

  job2:

    needs: job1

  job3:

    needs: [job1, job2]

作业包含一系列任务，称为 steps。 步骤可以运行命令、运行设置任务，或者运行您的仓库、公共仓库中的操作或 Docker 注册表中发布的操作。 并非所有步骤都会运行操作，但所有操作都会作为步骤运行。

steps

name: Greeting from Mona

on: push

jobs:

  my-job:

    name: My Job

    runs-on: ubuntu-latest

    steps:

      - name: Print a greeting

        env:

          MY_VAR: Hi there! My name is

          FIRST_NAME: Mona

          MIDDLE_NAME: The

          LAST_NAME: Octocat

        run: |

          echo $MY_VAR $FIRST_NAME $MIDDLE_NAME $LAST_NAME.

选择要作为作业中步骤的一部分运行的操作。 操作是一种可重复使用的代码单位。 可以使用在与工作流、公共存储库或已发布的 Docker 容器映像相同的存储库中定义的操作。。

uses

steps:

  # Reference a specific commit

  - uses: actions/checkout@8f4b7f8486

  # Reference the major version of a release

  - uses: actions/checkout@v4

  # Reference a specific version

  - uses: actions/checkout@v4.2.0

  # Reference a branch

  - uses: actions/checkout@main
